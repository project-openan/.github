# Getting Help with OpenAN

OpenAN is an open source project under Linux Foundation Networking. Here is how to get help.

## Community Resources

- **Mailing Lists:** [lists.openan.dev](https://lists.openan.dev) is the primary channel for discussions, questions, and announcements.
- **Wiki:** The [OpenAN Confluence wiki](https://lf-networking.atlassian.net/wiki) contains project documentation, meeting notes, and technical resources.
- **Website:** [project-openan.github.io](https://project-openan.github.io) for project overview and links.

## Reporting Bugs

If you have found a bug, please open an issue in the relevant repository with a clear description, reproduction steps, and any relevant logs.

## Feature Requests

Feature requests and ideas are welcome. Please propose them on the [mailing list](https://lists.openan.dev) or open an issue in the relevant repository.

## Security Issues

If you have found a security vulnerability, please do **not** open a public issue. Instead, follow the [Security Policy](SECURITY.md) for responsible disclosure.

## TSC Meetings

The OpenAN Technical Steering Committee (TSC) holds regular meetings open to all community members. Meeting schedules, agendas, and minutes are available on the [wiki](https://lf-networking.atlassian.net/wiki).

## Project Support

- **LFN Project Support:** For questions about governance, community operations, mailing lists, wiki access, or other project management topics, contact the LFN Technical Community Architect / Program Manager team at [support@lfnetworking.org](mailto:support@lfnetworking.org).
- **LF IT Support:** For infrastructure issues (GitHub access, SSO/LFID accounts, tooling problems), submit a request at [support.linuxfoundation.org](https://support.linuxfoundation.org).
